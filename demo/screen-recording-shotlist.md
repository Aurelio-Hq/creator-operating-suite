# Shotlist de gravação

## Sequência de Cenas

1.  **Cena 1: Introdução ao Problema (15s)**
    *   Gravar a tela mostrando um cenário comum de uso do Claude sem organização (ex: várias abas, prompts repetitivos).
    *   Narração: "Criadores usam IA todos os dias, mas ainda enfrentam a frustração de reexplicar contexto, copiar instruções e perder decisões entre chats."

2.  **Cena 2: Apresentação do GitHub (10s)**
    *   Gravar a tela mostrando a página inicial do repositório `Content-creator-Day-to-day-routine` no GitHub.
    *   Foco no README, mostrando a frase central: "Não é um prompt pack. É um sistema operacional de trabalho..."
    *   Narração: "Apresentamos o Creator Operating Suite, um sistema operacional de trabalho para criadores não-devs."

3.  **Cena 3: O Trigger (10s)**
    *   Gravar a tela abrindo o Claude Code (ou interface similar) e mostrando o arquivo `CMD_HANDS_OFF_PUBLIC_DEMO.md`.
    *   Narração: "Com um simples 'trigger', o Claude já sabe quem você é, o que você faz e como você quer as respostas."

4.  **Cena 4: O Workflow em Ação (30s)**
    *   Gravar a tela rodando o `CMD_HANDS_OFF_PUBLIC_DEMO.md`.
    *   Mostrar brevemente a execução, sem focar em detalhes técnicos do terminal.
    *   Focar no resultado: Claude gerando um output consistente e estruturado.
    *   Narração: "Nosso sistema carrega o contexto relevante, aplica o workflow correto e gera outputs consistentes, sem que você precise reexplicar nada."

5.  **Cena 5: O Output e Validação (20s)**
    *   Gravar a tela mostrando o output final gerado pelo Claude (ex: um card de ativo, um roteiro de conteúdo).
    *   Destacar a consistência e a organização do resultado.
    *   Narração: "Decisões, riscos e próximas ações são logados, garantindo rastreabilidade e reusabilidade. O mesmo workflow pode ser usado repetidamente."

6.  **Cena 6: Chamada para Ação (15s)**
    *   Gravar a tela mostrando o README do GitHub novamente, com o link para o vídeo e o CTA para o beta feedback form.
    *   Narração: "Transforme seu Claude de um chat em um sistema de trabalho repetível. Acesse o repositório, experimente e junte-se à nossa comunidade beta!"

## Observações

*   Manter a gravação fluida e dinâmica.
*   Evitar mostrar muitos detalhes técnicos ou arquivos complexos.
*   Focar na solução do problema do criador não-dev.
*   Duração total: Máximo 2 minutos.
