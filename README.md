# Creator Operating Suite

Claude workflows, skills, templates, and decision systems for non-technical creators.

Creator Operating Suite helps solo creators, consultants, educators, and small teams turn Claude from a chat interface into a repeatable work system: context, workflows, decisions, templates, and outputs organized in one operational structure.

This repository is the public starter layer of a broader product architecture called **PersonalizationOS**.

## The problem

Creators use AI every day, but still face the same operational friction:

- re-explaining context every session;
- copying old instructions manually;
- losing decisions across chats;
- producing inconsistent outputs;
- mixing ideas, workflows, assets, and strategy without a source of truth.

## The solution

Creator Operating Suite organizes Claude usage into layers:

| Layer | Folder | Function |
|---|---|---|
| L0 | `starter/creator-entry-skill` | First reusable skill |
| L1 | `starter/personalization-os` | Main system spine |
| L2 | `suites/cmd-execution-suite` | Repeatable execution commands |
| L3 | `suites/intelligence-routers` | Framework and decision routing |
| L4 | `suites/visual-editorial-tools` | Content and visual deliverables |
| L5 | `suites/context-engineering-proofs` | Embedded context demos |
| L6 | `validation` | Asset cards and scorecards |

## Start here

1. Read `creator-suite-commercial-thesis-brief.md`.
2. Open `starter/creator-entry-skill/SKILL.md`.
3. Review `examples/before-after-demo.md`.
4. Use `validation/creator-asset-cards.yaml` as an inventory model.

## Core thesis

The value is not in one prompt or one skill. The value is in a repeatable operating model:

> context → workflow → decision → template → output → validation.
